# Field Ops Portfolio

An interactive portfolio of five connected applications built to run technology
operations across a portfolio of luxury estates. Each app turned a manual,
reactive process into something automated and proactive.

Open **`index.html`** to start. It links to a working, in-browser demo of each app:

| App | What it is |
| --- | --- |
| **Beacon** | Always-on infrastructure status wall. Calm when healthy, impossible to miss when something breaks. |
| **Engine** | Monitoring-to-ticketing pipeline. Files and clears tickets automatically, one incident at a time. |
| **Recon** | Guided on-site technology field audit that files a structured record. |
| **Generator** | Pulls a visit's work and exports a premium, client-ready PDF report. |
| **Tempo** | Scores team adoption of the work platform against role benchmarks and auto-coaches. |

Also included: **Ambient**, an ideation-stage concept for passive field documentation.

## Notes

- Every demo runs entirely in your browser. No live systems, logins, or real data are connected.
- All names, properties, and figures are fictional; company and client specifics are intentionally generalized.
- Demos pull web fonts from Google, so they look best with an internet connection.
- Beacon, Recon, and Generator run the actual production front-end code with the live data layer swapped for mock data. Tempo is the real React app compiled with sample data.
